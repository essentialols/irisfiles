Privacy Policy

Last updated: February 2026

How IrisFiles works

IrisFiles is a client-side file converter. All file processing happens entirely in your web browser using JavaScript, WebAssembly, and the Canvas API. Your files are never uploaded to any server. No file data leaves your device at any point during the conversion process.

Data we collect

None. IrisFiles does not collect, store, transmit, or process any personal data or file data. There are no accounts, no cookies, no analytics, no tracking scripts, and no server-side logging of any kind.

Third-party resources

Some conversion tools lazy-load open-source libraries from cdn.jsdelivr.net (a public CDN). These requests are standard HTTP fetches for JavaScript and WebAssembly files. jsDelivr's own privacy policy applies to those network requests. No file data is sent to jsDelivr or any other third party.

Hosting

IrisFiles is hosted on Vercel as a static site. Vercel may log standard web server information (IP addresses, request timestamps, user agents) as part of normal CDN operation. IrisFiles has no access to or control over these logs. See Vercel's privacy policy for details.

Local storage

IrisFiles uses IndexedDB (browser-local storage) solely to pass files between pages when using the smart file dropper on the landing page. This data is temporary, stays on your device, and is deleted immediately after the target converter page reads it.

File conversions are performed using browser APIs and open-source libraries. Results may vary depending on your browser, operating system, and the specific files being converted. IrisFiles does not guarantee that converted files will be suitable for any particular purpose. Always keep your original files.

Large file warning

Converting large files (especially video, audio, and high-resolution images) happens entirely in your browser's memory. This means:

  • Large files may cause your browser tab to use significant memory (potentially several gigabytes for video conversions).
  • Your browser may become slow or unresponsive during conversion of very large files.
  • On devices with limited memory (phones, tablets, older computers), large conversions may fail or crash the browser tab.
  • IrisFiles enforces a 100MB file size limit and a 100-megapixel image limit as safeguards, but even files within these limits can strain low-memory devices.

For best results with large files, close other browser tabs, use a desktop computer, and ensure you have sufficient free memory.
